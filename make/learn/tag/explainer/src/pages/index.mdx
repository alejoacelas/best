## The whole thing in six substrates

- **Trigger surface** — Slack. An `@Claude` mention, a <Term id="routine">routine</Term>, or <Term id="ambient">ambient</Term> monitoring opens a task bound to the thread.
- **Execution surface** — a cloud Claude Code-style harness running the <Term id="agent-loop">agent loop</Term> inside the sandbox _(inferred; from the "evolution of Claude Code" framing + reporting)_.
- **Body** — the ephemeral per-thread sandbox: real code execution, discarded on idle, rebuilt on next reply.
- **Memory** — a persistent, admin-governed store scoped **per channel / per workspace** <Src src="memoryDoc"/>. Storage mechanism undisclosed — see [Open threads](/open-threads/).
- **Auth** — Agent Identity service accounts + Agent Proxy credential injection + default-deny egress.
- **Governance** — dual-level <Term id="spend-cap">spend caps</Term>, an audit console (every task, memory write, network call, user-attributed), scoped identities per function.

<Callout type="key">
A persistent, multiplayer principal whose body is a disposable cloud sandbox and whose state is a channel-scoped governed store. Everything downstream — the security model, the memory scoping, the governance surface — falls out of *state lives in the channel, compute is throwaway, and it reads untrusted input.*
</Callout>
